#READ

You will need to complete these 4 steps for the workflow to function correctly.





#### FIRST THING TO DO BEFORE RUNNING THIS WORKFLOW

First you will need to go into this directory:

/Complete_Cyclospora_typing_workflow_MacOS_High_Sierra_ALPHA_TEST/EUKARYOTYPING

In this folder you will see a .zip file called "REFERENCE_POPULATION.zip"

Unzip it. If you want to use these files as your reference population, thats fine. Alternatively, remove them and replace them with a reference of your choosing.



#### SECOND THING TO DO BEFORE RUNNING THIS WORKFLOW


You will need to go into this directory:

/Complete_Cyclospora_typing_workflow_MacOS_High_Sierra_ALPHA_TEST/HAPLOTYPE_CALLER_CYCLO_V2

In this folder you will see a .zip file called "SPECIMEN_GENOTYPES.zip"

Unzip it. This is where the genotype of every specimen will be written. This folder will contain the genotype of numerous specimens typed in 2019. You can remove these or you can practice running various modules of the workflow on these files.



#### THIRD THING TO DO BEFORE RUNNING THIS WORKFLOW

1. Open the "haplotype_sheets" folder, read the placeholder file, and then delete it.

2. Open the "ensemble_matrices" folder, read the placeholder file, and then delete it.

3. Open the "clusters_detected" folder, read the placeholder file, and then delete it.


### FOURTH THING TO DO BEFORE RUNNING THIS WORKFLOW

Install each of the dependencies in the folders within this directory;

/Complete_Cyclospora_typing_workflow_MacOS_High_Sierra_ALPHA_TEST/HAPLOTYPE_CALLER_CYCLO_V2

This will include;

- BBMAP
- BLAST
- BOWTIE
- CAP3
- CD-HIT (will need to be compiled in place)
- MIRA

In many cases, the required versions are provided within each of these directories within the HAPLOTYPE_CALLER_CYCLO_V2 directory. However, some software dependencies require a universal installation on your system. Please refer to the detailed installation instructions provided with this workflow for details. If you are missing even one of the haplotypes from the lists below in the files generated for these test specimens, then the workflow has not run correctly and you will need to double check that all dependencies are met.
